策略简介

Model 1 是一套以 Claude AI 为实时分析师的 BTC 波段交易系统，全栈运行于 Hetzner 德国 VPS。 通过多源信号融合 —— 实时市场快照、链上清算热力图、订单簿深度、OI 趋势、多源散度检测、宏观日历、分析师 YouTube 转录 —— 自动生成 3 至 5 条候选交易计划， 再交由本地状态机引擎执行入场、止损、分批止盈。

策略架构经过六个月持续迭代，目前包含：

  • 纯技术回测 → 实时 AI 分析师驱动的维度化交易计划设计；
  • 单向预测 → 多反弹原型与延续型套利两类共存（fade vs continuation 由 regime 决定）；
  • 固定百分比止损 → 基于 ATR + 结构层的动态止损（避开做市商常规猎杀区）；
  • 简单条件检查 → 维度分组规则（组内 OR + 组间 AND，触发组必满足 + 60% 多数即激活）；
  • 每日单次刷新 → 事件驱动（散度翻转 / $1500 价格异动 / ESCGO 极值穿越即时唤醒）；
  • 消息通知冗余 → 沉默模式（无关键变化时全静默，避免噪音）。

目前以 1 BTC 标准仓位 在模拟盘实时验证策略边际， 数据流：Binance 期货 + Coinglass API + Coinglass 热力图（headless Chrome）+ Coinalyze + Farside ETF + Bitcoin Hyper 视频转录。

🏗️ 系统架构

三个独立服务在 VPS 上常驻：

  • hyper_monitor · 5 秒轮询，评估候选计划条件，管理状态机，向 Telegram 推送状态变迁；
  • brain_watcher · Claude 分析大脑，事件驱动唤醒，重新设计/归档/调整候选计划；
  • brief_assembler · 每小时自动拼装市场简报（含散度 / 流量 / ETF / 清算 / 分析师视频）。

PC 端微任务：YouTube 转录 (yt-dlp + youtube-transcript-api) 与 Coinglass 热力图截取 (Playwright + React Fiber 读取)，每小时同步至 VPS。

设计原则：维度化交易 —— 不依赖单一指标，组合散度 + 结构 + 订单簿 + 流量 + 情绪；触发组 + 60% 多数满足即激活，避免过度过滤导致零执行。
